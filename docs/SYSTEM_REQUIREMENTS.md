## 最低要求

### 系统配置
| 组件 | 最低要求 |
|-----------|-------------------|
| **操作系统** | Windows 10 64位 (v1903+) 或 Ubuntu 18.04+ |
| **处理器** | Intel Core i3-4000 系列 / AMD FX-6000 系列 |
| **内存** | 4 GB RAM |
| **存储** | 50 GB 可用空间（推荐使用 SSD） |
| **网络** | 宽带互联网连接 |
| **显卡** | 兼容 DirectX 11（用于 GUI 模式） |

### 单个方舟服务器
- **CPU**: 双核，最低 2.5 GHz
- **内存**: 4 GB 系统内存 + 2 GB 用于方舟服务器
- **存储**: 25 GB 用于方舟服务器文件 + 10 GB 用于应用程序
- **网络**: 5 Mbps 上传（10 名玩家），每增加 5 名玩家需要 1 Mbps

## 推荐配置

### 最佳性能
| 组件 | 推荐配置 |
|-----------|-------------------------|
| **操作系统** | Windows 11 64位 或 Ubuntu 22.04 LTS |
| **处理器** | Intel Core i5-8400 / AMD Ryzen 5 2600 |
| **内存** | 16 GB RAM |
| **存储** | 100 GB SSD（首选 NVMe） |
| **网络** | 专用服务器连接（100+ Mbps） |
| **显卡** | 集成显卡足以满足 GUI 需求 |

### 多个方舟服务器（2-4 个服务器）
- **CPU**: 6+ 核，3.0+ GHz (Intel i7/Ryzen 7)
- **内存**: 24-32 GB（每个服务器 8 GB + 系统 8 GB）
- **存储**: 200+ GB SSD
- **网络**: 20+ Mbps 上传带宽

### 高性能设置（5 个以上服务器）
- **CPU**: 8+ 核，3.5+ GHz (Intel i9/Ryzen 9)
- **内存**: 48+ GB（每个服务器 8-12 GB）
- **存储**: 500+ GB NVMe SSD
- **网络**: 专用服务器连接（1 Gbps）

## 硬件要求

### 处理器 (CPU)
**最低**: 双核，2.5 GHz
- Intel: Core i3-4000 系列或更新
- AMD: FX-6000 系列或更新

**推荐**: 4+ 核，3.0+ GHz
- Intel: Core i5-8400 或更新
- AMD: Ryzen 5 2600 或更新

**企业级**: 8+ 核，3.5+ GHz
- Intel: Core i7-9700K, i9-9900K, 或更新
- AMD: Ryzen 7 3700X, Ryzen 9 3900X, 或更新

### 内存 (RAM)
**分配指南**:
- **系统开销**: 2-4 GB 用于操作系统
- **应用程序**: 1-2 GB 用于 Cerious AASM
- **每个方舟服务器**: 4-8 GB（取决于玩家数量和模组）

**玩家数量与内存映射**:
| 玩家 | 每个服务器内存 | 示例总计 (1个服务器) |
|---------|----------------|--------------------------|
| 1-10 | 4 GB | 系统总计 8 GB |
| 11-25 | 6 GB | 系统总计 12 GB |
| 26-50 | 8 GB | 系统总计 16 GB |
| 51-70 | 10 GB | 系统总计 20 GB |
| 70+ | 12+ GB | 系统总计 24+ GB |

### 存储
**类型要求**:
- **强烈推荐 SSD**: 方舟服务器极大地受益于快速的 I/O
- **可接受 HDD**: 用于备份存储和不太重要的数据
- **最佳 NVMe SSD**: 用于高性能设置

**空间分配**:
- **基础应用程序**: 2 GB
- **SteamCMD**: 1 GB
- **方舟服务器**: 25-30 GB（基础游戏 + 地图）
- **模组**: 每个服务器 1-10 GB（取决于模组选择）
- **存档数据**: 每个服务器 100 MB - 2 GB（随时间增长）
- **备份**: 为备份存储计划 2-5 倍的服务器大小

**示例存储规划**:
```
单服务器设置 (最低 50 GB):
├── 系统和应用程序: 10 GB
├── 方舟服务器文件: 25 GB
├── 存档数据和日志: 5 GB
└── 备份存储: 10 GB

多服务器设置 (推荐 200 GB):
├── 系统和应用程序: 20 GB
├── 方舟服务器文件 (3 个服务器): 75 GB
├── 共享模组存储: 15 GB
├── 存档数据和日志: 15 GB
└── 备份存储: 75 GB
```

### 显卡
**GUI 模式**: 
- 兼容 DirectX 11 的显卡
- 集成显卡即可
- 不需要独立显卡

**无头模式**:
- 没有显卡要求
- 可以在没有 GPU 的服务器上运行

## 软件依赖

### Windows 依赖
**自动安装**:
- Visual C++ Redistributable 2015-2022
- .NET Framework 4.8+
- DirectX Runtime

**可能需要手动安装**:
- Windows 更新（最新的累积更新）
- PowerShell 5.1+（用于防火墙管理）

### Linux 依赖
**Ubuntu/Debian**:
```bash
# 所需包
sudo apt update
sudo apt install -y curl wget ca-certificates gnupg lsb-release

# 用于 GUI 模式
sudo apt install -y libgtk-3-0 libgconf-2-4 libnss3 libxss1 libasound2

# 用于 AppImage
sudo apt install -y fuse
```

**CentOS/RHEL**:
```bash
# 所需包
sudo yum install -y curl wget ca-certificates

# 用于 GUI 模式
sudo yum install -y gtk3 nss alsa-lib
```

### 运行要求
- **Steam 账户**: 不需要（使用匿名登录）
- **方舟所有权**: 在服务器机器上不需要
- **互联网连接**: 初始设置和更新需要
- **管理员权限**: 安装和防火墙设置需要

## 网络要求

### 带宽指南
**每个服务器的带宽使用情况**:
| 玩家 | 所需上传 | 所需下载 |
|---------|----------------|-------------------|
| 1-10 | 2-5 Mbps | 1-2 Mbps |
| 11-25 | 5-10 Mbps | 2-3 Mbps |
| 26-50 | 10-20 Mbps | 3-5 Mbps |
| 51-70 | 20-35 Mbps | 5-8 Mbps |

### 端口要求
**所需端口（每个服务器）**:
- **游戏端口**: 27015 (TCP) - 玩家连接
- **查询端口**: 27015 (UDP) - Steam 服务器发现
- **RCON 端口**: 27020 (TCP) - 远程管理

**可选端口**:
- **Web 界面**: 3000 (TCP) - 默认 Web UI 端口
- **Steam Master**: 27017 (UDP) - Steam 服务器列表

**端口计算**:
```
服务器 1: 游戏 27015, 查询 27016, RCON 27020
服务器 2: 游戏 27025, 查询 27026, RCON 27030
服务器 3: 游戏 27035, 查询 27036, RCON 27040
(每增加一个服务器加 10)
```

### 防火墙配置
**需要入站规则**:
- 允许配置的游戏端口 (TCP)
- 允许配置的查询端口 (UDP)
- 允许 RCON 端口 (TCP) - 如果使用远程管理
- 允许 Web 界面端口 (TCP) - 如果使用 Web UI

**需要出站规则**:
- 允许 Steam 内容服务器（端口 443, 80）
- 允许 Steam master 服务器
- 允许 NTP 进行时间同步（端口 123）

## 存储要求

### 磁盘 I/O 性能
**最低 I/O 要求**:
- **顺序读取**: 100 MB/s
- **顺序写入**: 50 MB/s
- **随机读取 IOPS**: 500 IOPS
- **随机写入 IOPS**: 250 IOPS

**推荐 I/O 性能**:
- **顺序读取**: 500+ MB/s
- **顺序写入**: 300+ MB/s
- **随机读取 IOPS**: 2000+ IOPS
- **随机写入 IOPS**: 1000+ IOPS

### 文件系统要求
**Windows**:
- 需要 NTFS 文件系统
- 服务器文件不支持 ExFAT
- Windows Server 支持 ReFS

**Linux**:
- 推荐 ext4（经过最多测试）
- 支持 XFS（适合大文件）
- 支持 Btrfs（高级用户）
- 支持 ZFS（高级用户，需足够内存）

### 备份存储规划
**备份策略考虑**:
- **本地备份**: 相同的磁盘性能要求
- **网络备份**: 网络带宽成为限制因素
- **云备份**: 互联网上传速度限制
- **归档存储**: 可以使用更慢、更便宜的存储

## 性能扩展

### 单个服务器性能
**玩家数量与要求对比**:
```
10 玩家:  双核, 4GB 内存, 5 Mbps 上传
25 玩家:  4 核, 6GB 内存, 10 Mbps 上传
50 玩家:  4 核, 8GB 内存, 20 Mbps 上传
70 玩家:  6 核, 10GB 内存, 35 Mbps 上传
```

### 多个服务器扩展
**扩展考虑**:
- CPU 核心数随服务器数量线性扩展
- 内存要求每个服务器叠加
- 网络带宽每个服务器叠加
- 磁盘 I/O 成为共享资源（SSD 至关重要）

**多服务器配置示例**:
```
3 个服务器（每个 25 名玩家）:
├── CPU: 8 核, 3.0+ GHz
├── 内存: 24 GB (6GB × 3 个服务器 + 6GB 系统)
├── 存储: 100 GB SSD
└── 网络: 30 Mbps 上传

5 个服务器（每个 50 名玩家）:
├── CPU: 12 核, 3.5+ GHz
├── 内存: 48 GB (8GB × 5 个服务器 + 8GB 系统)
├── 存储: 200 GB NVMe SSD
└── 网络: 100 Mbps 上传
```

## 企业要求

### 生产环境
**高可用性设置**:
- **主服务器**: 主应用程序实例
- **备份服务器**: 热备用以进行故障转移
- **负载均衡器**: 分配 Web 界面访问
- **共享存储**: 用于服务器文件的网络附加存储

**监控要求**:
- **系统监控**: CPU、内存、磁盘、网络使用情况
- **应用程序监控**: 服务器状态、玩家数量
- **日志聚合**: 集中日志解决方案
- **警报**: 针对问题的自动通知

### 虚拟化支持
**VMware vSphere**:
- ✅ 完全支持
- 最少 2 vCPU, 8 GB vRAM
- 需要 SSD 支持的存储

**Hyper-V**:
- ✅ 支持
- 推荐第二代虚拟机
- 支持动态内存

**容器支持**:
- ⚠️ 有限支持
- Docker 容器可行但复杂
- Kubernetes 部署可行

### 云部署
**AWS EC2 推荐实例**:
- **单个服务器**: t3.large (2 vCPU, 8 GB 内存)
- **多个服务器**: c5.2xlarge (8 vCPU, 16 GB 内存)
- **高性能**: c5.4xlarge (16 vCPU, 32 GB 内存)

**Azure 虚拟机**:
- **单个服务器**: Standard_D2s_v3
- **多个服务器**: Standard_D4s_v3
- **高性能**: Standard_D8s_v3

**Google Cloud Platform**:
- **单个服务器**: n2-standard-2
- **多个服务器**: n2-standard-4
- **高性能**: n2-standard-8

## 兼容性说明

### 已知问题
**Windows 特定**:
- Windows Defender 可能会标记 steamcmd.exe（误报）
- 某些防病毒软件会干扰服务器文件
- Windows N 版本可能需要媒体功能包

**Linux 特定**:
- 某些发行版需要额外的库包
- SELinux 可能会阻止网络操作（可以配置）
- Wayland 显示服务器支持有限（推荐使用 X11）

### 版本兼容性
**方舟服务器兼容性**:
- 支持所有官方方舟服务器版本
- 新方舟版本发布时自动更新
- 与现有存档文件向后兼容

**操作系统更新**:
- 完全支持 Windows 功能更新
- Linux 内核更新通常兼容
- 主要操作系统升级可能需要更新应用程序

### 硬件兼容性
**CPU 架构**:
- 需要 x86_64 (64位)
- 不支持 ARM 处理器
- 支持虚拟 CPU 环境

**存储接口**:
- 支持 SATA SSD/HDD
- 支持并推荐 NVMe 驱动器
- 支持网络存储 (NFS, SMB)，但存在性能注意事项
- 不推荐将 USB 外部驱动器用于服务器文件

### 性能优化提示
1. **使用 SSD 存储**以显著提升性能
2. **分配充足的内存**以避免交换
3. **监控 CPU 使用率**，如果持续超过 80% 则进行升级
4. **定期维护**，包括磁盘碎片整理（仅限 HDD）
5. **网络优化**，如果可能的话通过 QoS 规则
6. **防火墙调优**以最小化延迟
7. **定期备份**以防止数据丢失
8. **更新管理**以确保安全和性能
